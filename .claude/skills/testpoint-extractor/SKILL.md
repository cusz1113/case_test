# 测试点提取技能 (Testpoint Extractor)

## 1. 技能定位

本技能专注于**阶段3：测试点提取** ，负责：

1. 按章节读取需求 JSON 文档及关联图片
2. **遍历章节JSON中每一个需求，将每个需求描述和图片一起给大模型**提取这个需求的测试点
3. 每个需求的测试点提取完成之后，都要更新任务清单，一个一个需求点进行提取，可以并发调用子agent来完成任务

**前置依赖**：requirement-splitter 技能已成功执行并输出 JSON 需求文档和图片文件

---

## 0. JSON 文件格式严格要求（最高优先级）

**生成任何 JSON 文件时必须严格遵守以下规则，违反即为不可接受的错误：**

1. **必须使用 `json.dumps()` 写入文件**：禁止手动拼接 JSON 字符串。所有 JSON 输出必须通过 Python 的 `json.dump(data, f, ensure_ascii=False, indent=2)` 或等价方式生成。

2. **中文引号规则**：JSON 字符串值内部的中文双引号必须使用全角引号 `“`（"）和 `”`（"），严禁使用 ASCII 双引号 `"`（"）出现在字符串值内部。

3. **写入后必须验证**：写入 JSON 文件后，必须立即用 `json.load()` 重新读取验证。

4. **常见错误清单**（写入前逐项检查）：
   - 字符串值内出现未转义的 ASCII `"`（最常见错误）
   - 字符串值内出现未转义的反斜杠 `\`
   - 尾随逗号（数组/对象最后一个元素后多余的 `,`）

> **违反以上任何一条规则的 JSON 文件视为不合格产物，必须重新生成。**

```

│  需求信息 (给大模型)                         │

│  需求ID：REQ-CH3-001                        │

## 5. 测试点提取规则

### 5.1 测试点类型

| 类型     | 代码 | 说明                         |
| -------- | ---- | ---------------------------- |
| 功能测试 | FT   | 验证功能是否符合需求         |
| 边界测试 | BT   | 测试边界值条件               |
| 异常测试 | ET   | 测试异常情况和错误处理       |
| 性能测试 | PT   | 测试性能指标                 |
| UI测试   | UIT  | 测试界面交互（**基于图片**） |
| 安全测试 | ST   | 测试安全相关功能             |

### 5.2 优先级定义

| 优先级 | 说明               | 适用场景                 |
| ------ | ------------------ | ------------------------ |
| P0     | 核心功能、基本流程 | 必须通过，否则功能不可用 |
| P1     | 重要功能、常用场景 | 应该通过，影响主要流程   |
| P2     | 一般功能、特殊场景 | 可以尝试，不阻塞上线     |
| P3     | 边缘功能、低频场景 | 可选                     |

### 5.3 图片关联测试点提取

**重要**：当需求中包含关联图片时，测试点提取需额外考虑：

| 图片类型   | 提取的测试点                                   |
| ---------- | ---------------------------------------------- |
| 流程图     | **流程测试**：验证功能流程与图中一致           |
| 界面截图   | **UI测试**：验证界面元素、布局、交互与截图一致 |
| 状态机图   | **状态转换测试**：验证状态转换符合图示         |
| 架构图     | **配置测试**：验证配置与架构一致               |
| 数据示例图 | **数据验证测试**：验证数据展示与图示一致       |

**图片测试点提取示例**：

```json
{
  "testpoint_id": "TP-CH3-001-010",
  "testpoint_name": "验证订票流程与流程图一致",
  "test_type": "UI测试 (UIT)",
  "priority": "P1",
  "preconditions": ["已登录B2C系统"],
  "test_steps": [
    "1. 参考 images/3a1d746c.jpg 流程图",
    "2. 按流程图步骤执行：选择出发地→选择目的地→选择航班→填写信息→支付",
    "3. 验证每个步骤与流程图一致"
  ],
  "expected_results": [
    "流程图中的5个步骤均能正常完成",
    "无流程图之外的额外步骤要求"
  ],
  "related_images": [
    "images/3a1d746c.jpg"
  ],
  "notes": "此测试点根据关联的流程图（images/3a1d746c.jpg）提取，确保实际流程与设计流程一致"
}
```

## 6. 验收标准到测试点的映射

### 6.1 映射规则

每个验收标准 **至少** 对应 **1 个测试点**：

```
验收标准 1 → 测试点 1
验收标准 2 → 测试点 2
验收标准 N → 测试点 N + 图片关联测试点（如有图片）
```

## 8. 执行流程

```
1. 解析输入参数
   ├── 验证 requirements_json 是否存在
   ├── 验证 output_dir 是否有效
   └── 验证 images/ 目录是否存在

2. 读取需求文档
   ├── 解析 requirements.json
   ├── 解析 requirements_detail.json（如有）
   └── 构建需求结构树

3. 图片加载
   └── 对每个需求：
       ├── 读取 related_images 中的图片文件
       └── 将图片内容加载到内存

4. 遍历需求并提取测试点
   └── 对每个需求执行：
       ├── 组装"需求描述+图片"联合输入
       ├── 调用大模型提取测试点
       ├── 解析大模型返回的JSON
       ├── 校验测试点完整性
       └── 保存测试点 JSON

5. 生成汇总
   ├── 生成各章节 testpoints.json
   ├── 生成全局汇总 testpoints_summary.json
   ├── 生成 README.md（索引）
   ├── 更新任务清单
   └── 输出执行报告
```

## 9. 错误处理

| 错误情况       | 处理方式                                    |
| -------------- | ------------------------------------------- |
| 需求JSON不存在 | 报错退出，提示先执行 requirement-splitter   |
| 图片文件缺失   | 继续处理，但测试点 notes 中标注"图片缺失"   |
| JSON格式异常   | 跳过异常需求，记录日志，继续处理其他需求    |
| 验收标准为空   | 生成至少1个通用功能测试点                   |
| 大模型调用失败 | 重试3次，仍失败则使用规则引擎生成基础测试点 |
