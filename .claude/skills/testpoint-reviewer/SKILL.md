# 测试点评审技能 (Testpoint Reviewer)

## 1. 技能定位

本技能专注于**阶段5：测试点评审**工作，负责：

1. 读取测试点 JSON 和需求 JSON 文件
2. 以资深软件测试工程师的角度，对测试点进行**全面质量评审**
3. 分析测试点是否覆盖了需求中明确提及的所有功能点
4. 输出评审报告，指出遗漏、问题和建议

**前置依赖**：testpoint-extractor 技能已成功执行并输出 testpoints.json

---

## 0. JSON 文件格式严格要求（最高优先级）

**生成任何 JSON 文件时必须严格遵守以下规则，违反即为不可接受的错误：**

1. **必须使用 `json.dumps()` 写入文件**：禁止手动拼接 JSON 字符串。所有 JSON 输出必须通过 Python 的 `json.dump(data, f, ensure_ascii=False, indent=2)` 或等价方式生成。

2. **中文引号规则**：JSON 字符串值内部的中文双引号必须使用全角引号 `“`（“）和 `”`（”），严禁使用 ASCII 双引号 `"`（"）出现在字符串值内部。

3. **写入后必须验证**：写入 JSON 文件后，必须立即用 `json.load()` 重新读取验证。

4. **常见错误清单**（写入前逐项检查）：
   - 字符串值内出现未转义的 ASCII `"`（最常见错误）
   - 字符串值内出现未转义的反斜杠 `\`
   - 尾随逗号（数组/对象最后一个元素后多余的 `,`）

> **违反以上任何一条规则的 JSON 文件视为不合格产物，必须重新生成。**

## 3. 评审维度与检查项

### 3.1 覆盖率分析（核心）

| 检查项              | 说明                                              |
| ------------------- | ------------------------------------------------- |
| 验收标准覆盖率      | 每个验收标准是否都有对应的测试点                   |
| 功能点覆盖率        | 需求描述中的每个功能点是否都有测试点覆盖           |
| 测试类型完整性      | 是否包含必要的 FT/BT/ET 等测试类型                 |
| 优先级合理性        | P0/P1/P2 分配是否与功能重要性匹配                 |

### 3.2 测试点质量评审

| 检查项              | 说明                                              |
| ------------------- | ------------------------------------------------- |
| 测试步骤完整性      | 测试步骤是否清晰、可执行、完整                     |
| 预期结果明确性      | 预期结果是否明确、可验证                           |
| 前置条件合理性      | 前置条件是否完整且可达成                           |
| 测试数据充分性      | 测试数据是否足够覆盖测试场景                       |
| 测试点可执行性      | 测试点是否可以在实际环境中执行                     |

### 3.3 遗漏检测

| 检查项              | 说明                                              |
| ------------------- | ------------------------------------------------- |
| 边界条件遗漏        | 是否有应该测试的边界值没有覆盖                   |
| 异常场景遗漏        | 是否有应该测试的异常情况没有覆盖                   |
| 关联场景遗漏        | 是否有跨模块/跨功能的关联场景没有覆盖               |

### 3.4 一致性检测

| 检查项              | 说明                                              |
| ------------------- | ------------------------------------------------- |
| 命名一致性          | 测试点命名是否与需求功能点命名一致                 |
| 字段完整性          | 测试点 JSON 字段是否完整                           |
| 编号连续性          | 测试点编号是否连续无跳跃                           |


   ├── 验证 testpoints_json 是否存在

   ├── 覆盖率分析

   ├── 生成 review_report.json
   ├── 生成 review_report.md

   └── 保存到 output_dir

   └── 向用户展示评审结果摘要
